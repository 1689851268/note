# HTTP

-   HTTP 协议，Hyper Text Transfer Protocol (超文本传输协议)，用于 [服务器] → [浏览器] 的数据传输

-   HTTP 基于 TCP / IP 通信协议 ("三握四挥") 来传递数据

-   默认端口：HTTP - `80`；HTTPS - `443`

> #### `Content-type`

标头的 `Content-type` 字段，可以指定内容的格式 (json、jpg、mp4...)，让浏览器更好地处理数据

# HTTP 2.0

> #### HTTP 2 相对于 HTTP 1 的改进

1. **头部压缩** (HPACK 技术) ：建立索引表，减少需要重复传输的头部信息
2. **分流**：把数据分割为更小的 frame 帧，并对其进行二进制的编码
3. **多路复用**：单个 TCP 链接可以解决多次 HTTP 请求，不会因为某个包的延迟而导致整体链路的阻塞
4. 对于用户可能使用到的静态资源，服务器可以**主动推送**给客户端 (Server Push)，客户端可以缓存这些资源

# HTTPS

-   HTTPS 基于 HTTP 协议，通过 SSL (Security Socket Layer, 安全套接层协议) / TLS (Transport Layer Security, 安全传输层协议) 
    ① 加密数据、 ② 验证通信方身份、 ③ 保证数据的完整性
-   HTTPS 提供对服务器的身份认证、对数据进行加密、保证数据的完整性
-   TLS 是传输层的加密协议，前身是 SSL 协议

> #### HTTP vs HTTPS

HTTP 的特点：

1. **无状态**：不会存储客户端的一些操作
2. **无连接**：由于无状态，每次请求都需要通过 TCP "三握四挥"，和服务器重新建立连接，会造成不必要的开销
3. 使用**明文**通信、请求和响应**不会确认通信方的身份**、无法保证数据的**完整性**

HTTPS 的特点：

1. 内容加密：采用混合加密技术，中间者无法直接查看明文内容
2. 验证身份：通过证书认证客户端访问的是自己的服务器
3. 保护数据完整性：防止传输的内容被中间人冒充 / 篡改

混合加密：结合 [非对称加密] & [对称加密]
先使用对称加密，生成密钥；然后使用非对称加密的公钥，再对密钥进行加密

# HTTP 状态码

HTTP 状态码由 3 个十进制数字组成
第 1 个数字定义了状态码的类型：

-   `1`-普通信息响应：服务器收到请求，需要请求者继续执行操作
-   `2`-成功响应：操作被成功接收并处理
-   `3`-重定向：需要进一步操作，以完成请求
-   `4`-客户端错误：请求包含语法错误 / 无法完成请求
-   `5`-服务器错误：服务器在处理请求的过程中发生了错误

# 面试题

## 301、302、307 有什么区别

-   `301`：永久重定向(Moved Permanently)，该操作比较危险，需要谨慎操作：如果设置了 301，但是一段时间后又想取消，但是浏览器中已经有了缓存，还是会重定向
-   `302`：临时重定向(Found)，但是会在重定向的时候改变 `method`：把 `POST` 改成 `GET`。于是有了 `307`
-   `307`：临时重定向(Temporary Redirect)，在重定向时不会改变 `method`

<br>

## 502、504 有什么区别

这 2 种异常状态码都与网关 Gateway 有关，首先明确两个概念：

-   Proxy (Gateway)：反向代理层 / 网关层。一般是 Nginx 充当
-   Application (upstream serrver)：应用层服务，作为 Proxy 层的上游服务。一般为各种语言编写的服务器应用

502 和 504 的区别：

-   `502`(Bad Gateway)：一般表现为自己写的应用层服务挂了，网关层无法接收到响应 (也有说 "收到了上游响应但无法解析" 的)
-   `504`(Gateway Timeout)：一般表现为应用层服务超时，即 "上游响应超时"
