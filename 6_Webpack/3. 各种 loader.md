# loader 加载器

- 在开发过程中，webpack 默认只能处理 .js .json 文件
- 对于其他文件，需要使用 loader 加载器才可以对其进行构建

# 构建过于高级的 js 语法

webpack 只能处理一部分高级 JS 语法，对于那些处理不了的 JS 语法，需要使用 babel-loader 进行构建

1. `pnpm i babel-loader @babel/core @babel/piugin-proposal-class-properties -D`
2. 配置 webpack.config.js

```js
const HtmlWebpackPlugin = require("html-webpack-plugin");

module.exports = {
    mode: "development",
    plugins: [new HtmlWebpackPlugin()],
    module: {
        rules: [
            {
                test: /\.js$/,
                exclude: /node_module/, // 可以使用 exclude 设置需要排除的文件(夹)
                use: {
                    // 配置 babel-loader
                    // 因为 @babel/core 是 babel-loader 的内置依赖项，所以需要下载但不需要配置
                    loader: "babel-loader",
                    options: {
                        // 配置插件，插件 @babel/plugin-proposal-class-properties 可用于处理 class 中的高级语法
                        plugins: ["@babel/plugin-proposal-class-properties"],
                    },
                },
            },
        ],
    },
};
```

3. 在入口文件 ./src/index.js 中编写 JS 的高级语法

```js
class Person {
    // 通过 static 定义静态属性
    // webpack 无法处理 "静态属性" 这个高级语法
    static info = "Person info";
}
console.log(Person.info);
```

5. 执行脚本 `pnpm open`，可以在浏览器中看到，`Person.info` 被成功打印出来了

   执行脚本 `pnpm build`，查看打包出来的文件：index.html + main.js
